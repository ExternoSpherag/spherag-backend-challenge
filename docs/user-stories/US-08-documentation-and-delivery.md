# US-08 — Documentation and Delivery

## Context
The final solution must be easy to run, review, and evaluate.

## Goal
Document setup, behavior, design decisions, and submission details.

## Acceptance Criteria
1. `README.md` explains setup, run, test, and API usage.
2. Binance stream URL and payload examples are documented.
3. API request/response examples (success + error) are documented.
4. Edge-case handling decisions are documented (duplicates, late events, empty windows).
5. PR title follows challenge format: `[Challenge] Your Name`.

## Notes for Implementation
- Keep documentation in English.
- Keep examples consistent with implemented endpoint routes.
